## 1. 概要

本ドキュメントは、Campus Connectアプリケーションにおける、BLE（Bluetooth Low Energy）を用いたすれ違い検知機能の技術仕様を定義する。
本機能の目的は、アプリケーションをインストールしたユーザー同士が物理的に接近したことを、プライバシーを保護しつつ、低消費電力で検知し、サーバーに記録することである。

## 2. 実装方針

Flutterの標準的な方法である **Platform Channels** を用いて、Flutterとネイティブコード（iOS: Swift, Android: Kotlin）を連携させる。
BLEのコアな処理（アドバタイズ、スキャン）はネイティブ側で実装し、Flutter側はその結果を受け取ってUIに反映させ、サーバーへの情報送信を担う。

## 3. 一時ID (Temporary ID, TID) とアドバタイズ

### 3.1. TIDの仕様
ユーザーのプライバシーを保護するため、個人に紐づく永続的なIDを直接BLEで発信するのではなく、15分ごとに自動で更新される一時的なID（TID）を使用する。

- **フォーマット**: UUIDv4
- **有効期間**: 15分
- **発行**: TIDはサーバーサイドで発行される。クライアント（アプリ）はログイン時や定期的な通信の際に、サーバーから数時間〜1日分のTIDリストを取得し、安全な領域に保管する。

### 3.2. アドバタイズ
- **内容**: 現在の時刻に有効なTIDを、BLEのアドバタイズパケットのService Dataフィールドに含めて発信する。
- **頻度**: OSの推奨する標準的な間隔（例: 200ms〜500ms）でアドバタイズを行う。

## 4. スキャンとデータ収集

### 4.1. スキャンポリシー (バランス重視)
バッテリー消費と検知精度のバランスを取るため、以下のポリシーでスキャンを実行する。

- **スキャン間隔**: **5分ごと**にスキャンを実行する。
- **スキャン時間**: 1回のスキャンは**10秒間**継続する。
- **動作**: アプリがフォアグラウンド、バックグラウンドの状態によらず、このスキャンサイクルを維持する。

### 4.2. データ収集
- スキャン中に検知した、本アプリが定義するService UUIDを持つデバイスからTID、RSSI、タイムスタンプを収集し、1回のスキャンサイクルが完了するまでデバイスのメモリに保持する。

## 5. データ形式 (ネイティブ-Flutter間)
スキャンが完了した後、ネイティブコードからFlutterへは、以下のJSONフォーマットにシリアライズされたオブジェクトのリストとしてデータを渡す。これにより、豊富な情報を活用でき、将来の機能拡張（距離の推定など）やデバッグに対応する。

- **形式**: リッチなオブジェクトリスト
- **例**:
  ```json
  [
    {
      "tid": "xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx",
      "rssi": -58,
      "timestamp": 1678886400
    },
    {
      "tid": "yyyyyyyy-yyyy-4yyy-zyyy-yyyyyyyyyyyy",
      "rssi": -65,
      "timestamp": 1678886405
    }
  ]
  ```
- **フィールド定義**:
  - `tid` (String): 検知した一時ID。
  - `rssi` (Integer): 受信信号強度。距離の推定に使用できる。
  - `timestamp` (Integer): 検知したUNIXタイムスタンプ（秒）。

## 6. エラーハンドリングとUX

ユーザーが機能を利用できない状況に陥らないよう、以下の通り丁寧なハンドリングを行う。

### 6.1. BLE利用権限が許可されない場合 (状況に応じた誘導)
- **トリガー**: ユーザーが「すれ違い」機能を利用しようとした（例:「すれ違い」タブを開くなど）にも関わらず、BLEの利用権限がない場合。
- **挙動**: アプリは以下の要素を含むUI（ダイアログまたは画面中央のメッセージ）を表示する。
  1.  **説明**: 「すれ違い機能の利用には、Bluetoothの権限が必要です。」といった明確なメッセージ。
  2.  **アクション**: OSの「設定」アプリの該当ページへ直接遷移させるボタン。
- **目的**: ユーザーが機能を「使いたい」と思ったタイミングで権限の必要性を伝え、許可を自然に促す。

### 6.2. スマートフォンのBluetoothがオフの場合 (機能の無効化表示)
- **トリガー**: アプリの利用中に、スマートフォンのBluetooth設定がオフになった場合。
- **挙動**: 「すれ違い」タブの画面全体をグレイアウト、または情報表示エリアを差し替えて、以下の内容を表示する。
  1.  **状態表示**: 「すれ違い機能がオフになっています。」
  2.  **理由と対処法**: 「機能を再開するには、コントロールセンターまたは設定からBluetoothをオンにしてください。」という案内メッセージ。
  3.  **アクション**: （可能であれば）OSの設定画面へ誘導するリンク。
- **目的**: 機能が使えない理由と対処法を一目瞭然にすることで、ユーザーの混乱を防ぐ。

## 7. サーバーとの連携とユーザー識別

BLE通信で収集したTIDは、以下のフローを経てサーバー側でユーザーの遭遇記録に変換される。

1.  Flutterは、ネイティブコードから受け取ったTID情報のリストを、自身の永続ID（User ID）と共に、サーバーの`/encounters`エンドポイントにPOSTする。
2.  サーバーは、リクエストされた永続IDを「遭遇した側」のユーザーとして認識する。
3.  サーバーは、リクエストボディに含まれるTIDのリストを、自身のデータベースに保持する「TIDと永続IDのマッピングテーブル」と照合する。
4.  照合に成功したTIDを「遭遇された側」のユーザーとして特定し、「誰が」「誰と」すれ違ったかの遭遇記録をデータベースに作成する。
